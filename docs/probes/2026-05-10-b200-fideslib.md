This kernel uses encrypted per-token inputs and encrypted recurrent state, with
public plaintext weights. The immediate bottleneck is not GPU throughput but
the naive level alignment strategy: 28 of 44 ciphertext-plaintext multiplies in
the default run are unity multiplies inserted only to align fresh encrypted
inputs with the recurrent state level. The next implementation step should make
level-aware input/update encoding explicit instead of paying this alignment
tax.
